# Stop AI Slop · 中文版

中文 AI 写作有它自己的指纹，跟英文不一样。本 Skill 不是英文版的翻译，是为中文重新做的。

## 一、写之前先要明白：中文 AI 味是什么

中文 AI 文本最容易暴露身份的不是单个词，是**整篇的气质**：

- 段段都有金句，每一段都试图"升华"
- 三件套排比反复出现："是…，是…，更是…"
- 名词化动词成灾："进行优化"、"做出贡献"、"实现突破"
- 抽象的东西当主语："时代呼唤"、"科技改变"、"AI 赋能"
- 总分总八股骨架明显，"首先 / 其次 / 综上"密集
- 价值判断饱和，什么都"重要"、"关键"、"核心"
- 假亲切，"朋友们 / 家人们 / 宝子们 / 让我们一起"
- 程度副词刷屏："非常 / 十分 / 极其 / 真的 / 其实"

这些不是单点问题，是一整套 AI 训练数据带来的"中文八股化"惯性。改写的核心不是替换词，是**拆掉骨架**。

往底层看，AI 味有两个可量化的特征：**困惑度低**（用词高度可预测，下一个字总挑最稳的那个）和**突发性低**（句子长短几乎不变，永远中正平稳）。人写东西会长短句交错、偶尔语序倒装、时不时冒一句口语——这种"不平稳"正是人味。本 Skill 后面的"变"维（句长变化）量的就是突发性；"直 / 实"维对治的是困惑度那种"安全到没有信息"的平滑。

## 二、九条核心改写规则

> ⚠️ **铁律：去具体 ≠ 编具体。** 下面规则 7 让你"把抽象换成数字 / 人名 / 场景"，但前提是**那个细节真实存在**。如果你（或模型）手上没有真实数据，**绝对不许编一个**——AI 幻觉出的"4.2 秒→0.8 秒""涨了 2000 粉"比套话更危险，因为它看起来像真的。手上没有真实细节时，三个合法选择：
> 1. 退回平实陈述（"加载明显变快了"虽然弱，但不撒谎）
> 2. 留占位符 `[作者补充：具体数字]`，交回作者填
> 3. 直接删掉这句空话
>
> 改写他人稿件时还要守：**不增事实、不改立场、不动语气倾向**。本 Skill 的职责是去 AI 味，不是替作者编内容。文档里的 Before/After 示例是"如果你有这个细节，长这样"的示范，不是"可以瞎编一个细节"的授权。

### 规则 1：删虚词和程度副词

删掉这些词后句子如果意思没变，那就该删：

- 程度副词：非常、十分、极其、相当、特别、格外、尤其、真的、确实、的确、实在
- 虚化量词：一种、某种、一定的、相应的、一定程度上、在某种意义上
- 模糊语气词：其实、本质上、说白了、客观来讲、坦白说

完整词表见 [references/phrases.md](references/phrases.md)（第 5 类：程度副词 / 虚词泛滥）。

> 原句：这其实是一种非常重要的能力。
> 改写：这是关键能力。

### 规则 2：拆排比三件套

中文 AI 病灶最重的一条。一旦看到 "A 是…，B 是…，C 是…" 或 "既要…又要…还要…"，直接拆。

> 原句：写作是一种思考，是一种表达，更是一种修行。
> 改写：写作就是思考。

排比把意思稀释了三遍——保留一句最有力的，其他扔掉。如果三句意思真的不同，分成三段或换成不同句式说。更多变体见 [references/structures.md](references/structures.md)（第 1 类：三件套排比）。

### 规则 3：去名词化

中文 AI 喜欢把动词变成"动词 + 名词"组合，听起来正式，其实是废话。

| AI 写法 | 改写 |
|---|---|
| 进行优化 | 优化 |
| 做出选择 | 选 |
| 实现增长 | 涨了 |
| 采取措施 | 怎么做（写出具体动作） |
| 起到作用 | 起作用 / 直接说做了什么 |
| 加以改进 | 改进 |
| 给予支持 | 支持 |
| 提供保障 | 保障 |

凡是"动词 + 抽象名词"的组合，先问能不能直接用那个动作的本词。

### 规则 4：删金句收尾、删抒情扩散

每段都想"升华"是 AI 的典型癖好。看到下面这类句子，删：

- "这，就是 XX 的力量"
- "唯有 XX，方能 XX"
- "因为 XX，所以 XX 才显得弥足珍贵"
- "在这个 XX 的时代，我们更需要 XX"
- "让我们一起，向着 XX 前进"

人写东西不会段段升华。段落自然结束就够了。

### 规则 5：用具体的人当主语，不用抽象主语

AI 喜欢用 "时代 / 科技 / AI / 未来 / 行业 / 现实" 这类抽象主语做事，因为这样不用交代到底是谁干的。

> 原句：时代的浪潮要求我们不断学习。
> 改写：我每周抽三小时学新东西，不然两年就跟不上了。

> 原句：AI 正在重塑我们的工作方式。
> 改写：我们组三个工程师，过去半年有两个把日常调试交给了 Claude。

抽象主语 = 信息密度为零。换成具体的人 / 具体的数字 / 具体的场景，文本才有重量。

### 规则 6：删元评论和八股连接词

中文 AI 喜欢提前预告自己要干什么，喜欢用连接词搭骨架。这些都删：

- 元评论：让我们来探讨 / 接下来我将 / 在本文中 / 下面我们来看 / 让我为你详细讲解 / 以下是几点
- 八股连接：首先 / 其次 / 再者 / 最后 / 综上所述 / 总而言之 / 由此可见 / 总的来说 / 概括而言
- 强调虚壳：值得注意的是 / 值得一提的是 / 不得不说 / 毫无疑问 / 不容忽视 / 我们必须认识到
- 无名归因（黄鼠狼话术）：专家认为 / 研究表明 / 数据显示 / 有人说 / 业内普遍认为——给不出实名来源就是在糊弄。**要么写出具体的人名、机构、文献，要么删掉这句。**
- AI 口头禅：融入了 / 结合了 / 这件事的本质是 / 说到底 / 我必须诚实地说 / 这个问题是真实存在的——后两条是英文 "I must be honest" / "this is a real concern" 的直译腔，看到即删。

直接进入内容。一个段落开始就讲事，不用预告这段要讲什么。

### 规则 7：给具体——数字、场景、人名、动作

价值判断词（重要、关键、核心、显著、巨大、深远）是 AI 的拐杖。把它们换成具体的东西。

> 原句：这次改进显著提升了用户体验。
> 改写：加载从 4.2 秒降到 0.8 秒，客服上周收到的"卡顿"投诉从 30 多条变成 2 条。

> 原句：我们的产品具有很强的竞争力。
> 改写：同价位里只有我们做了离线模式，对地铁通勤用户有用。

模糊词每出现一次，就问一遍：能换成具体吗？**但记住开头那条铁律——能换成"真实的具体"才换，没有就退回平实陈述或留占位符，不许编。**

### 规则 8：砍八股结构

不要总分总。不要每段都开一个标题概念然后展开然后小结。

中文 AI 最常见的篇章骨架：

```
开篇定义/背景 → 分点 1 → 分点 2 → 分点 3 → 总结升华
```

这是高中议论文结构，不是给人读的文章结构。可替代的骨架：

- **直接开干**：第一句就是核心结论或具体场景，后文是论据
- **故事线**：按时间或冲突推进，不分点
- **倒三角**：最反常识的放最前面，往后越来越细
- **单点深挖**：只讲一件事，把它讲透

如果文章一定要分点，至少不要每点都用"首先/其次/最后"开头，每点的开篇句要长得不一样。

**还要查标题层**：长文 / 技术文档正文再干净，如果一排小标题都用同一个模子（都是"名词：一句点题"、都是反问句、都是"为什么是 A 不是 B"），目录扫一眼就像机器列的提纲。把标题抄到一起看，超过一半同句式就打散——冒号副标题留一两个，反问标题最多一个。详见 [references/structures.md](references/structures.md)（第 16 类：标题套路）。

### 规则 9：管标点

中文 AI 有自己的标点偏好，最明显的是**破折号滥用**——这是英文 em-dash 的直接迁移，用来制造补充说明或戏剧性停顿。

> 原句：这个数字——远超行业平均——让所有人吃惊。
> 改写：这个数字远超行业平均，让所有人吃惊。

其他标点信号：

- **破折号「——」**：十有八九能换成逗号或直接删，整篇出现超过 1 次就要警惕
- **省略号「……」**：用来装"欲言又止"，多半是没话找话，删
- **感叹号「！」**：情绪是写出来的不是标点喊出来的，一段最多一个
- **双引号刷概念**：满篇`"赋能""闭环""抓手"`，加引号≠有内涵，删引号后读读看

冒号「：」本身没问题（列举、引语都正常用），只是别用它来给每段"起标题然后展开"，那是八股结构（见规则 8）不是标点问题。完整规则见 [references/punctuation.md](references/punctuation.md)。

## 三、五维评分量规

写完后给文本打分。每维 1–10，总分低于 **35/50** 必须重写。

| 维度 | 关注什么 | 1 分（差） | 10 分（好） |
|---|---|---|---|
| **直** | 直白度，虚词密度 | 满篇"其实/一种/进行/非常"，绕来绕去 | 一句一个意思，删任何字都损失信息 |
| **实** | 具体度，信息密度 | 全是抽象概念、价值判断、空泛形容词 | 有名字、数字、动作、场景，能在脑里画出画面 |
| **变** | 节奏，句长变化 | 句子长度差不多，排比三件套反复 | 长短交错，节奏自然，无明显套句 |
| **散** | 去骨架，去八股 | "首先/其次/综上"密布，总分总明显 | 看不出框架，但文意清楚 |
| **真** | 人味，视角 | 没人在说话，全是抽象主语，金句收尾 | 一个具体的人在讲他知道的事，有判断有态度 |

打分时严苛一点，**不确定就给低分**。35 分线不是"还行"线，是"勉强能发出去"线。低于 35 重写，35–42 修补，43+ 才算干净。

## 四、交付前清单

在按"发送"之前，过一遍：

- [ ] 数一下"非常/十分/真的/其实/进行/一种"出现次数，每千字不超过 3 个
- [ ] 全文搜 "首先/其次/再者/最后/综上/总而言之"，全部删或换
- [ ] 搜 "进行/实现/做出/采取/加以/给予" + 名词的组合，去名词化
- [ ] 找抽象主语（时代/科技/AI/未来/行业 + 动词），换成具体的人
- [ ] 找排比三件套（三个并列短句结尾相同结构），拆
- [ ] 把所有小标题抄到一起看，超过一半同句式（全冒号/全反问/全对立）就打散
- [ ] 找段尾的"金句"（这就是…的力量 / 唯有…方能…），删
- [ ] 数破折号「——」，全文超过 1 个就逐个看能不能换逗号或删；省略号、感叹号同理
- [ ] 找假亲切（朋友们/家人们/宝子们/亲爱的读者），删
- [ ] 找"值得注意的是 / 不得不说 / 毫无疑问"，删
- [ ] 搜"专家认为 / 研究表明 / 数据显示"无名归因，补实名来源或删
- [ ] 搜英译洋腔词（深入研究 / 见证 / 格局 / 强调 / 错综复杂 / 充满活力），换回普通中文
- [ ] **回查每个"具体数字 / 人名 / 场景"是不是真的**——凡是改写时填进去的，确认有出处，编的一律退回平实陈述或留 `[作者补充]`
- [ ] 五维打分，<35 重写

## 五、什么时候不该用本 Skill

这套规则面向**通用中文写作**——公众号、小红书、知乎、产品文案、邮件、博客、报告摘要。但有几个场景要谨慎：

- **正式公文 / 党政文稿**：里面的"总分总"、"首先其次"、价值判断词是结构要求，不是 AI 味
- **法律 / 合同文本**：精确性优先，名词化和长定语有时是必要的
- **学术论文（部分句式）**：被动语态、抽象主语在学术写作里是常态
- **小红书 / 抖音文案**：本身就是高度套路化的格式，需要的是"换一种套路"而不是去套路化

碰到这些场景，把规则当成参考而不是铁律。重点抓 1 / 3 / 5 / 7（删虚词、去名词化、用人当主语、给具体）。
