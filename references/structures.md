# 中文 AI 句式套路黑名单

词汇是表层，句式是深层。AI 即使把禁用词全换掉，骨架不变还是一眼能认出。这份清单针对句式和篇章结构。

---

## 1. 三件套排比（中文 AI 最强指纹）

把同一个意思用三个相同结构的短语说三遍，是 AI 写中文时最稳定的发病点。

**典型模板**：
- "A 是 X，B 是 X，C 是 X"
- "既要 A，也要 B，还要 C"
- "为了 A，为了 B，也为了 C"
- "是 A，是 B，更是 C"
- "不仅 A，还 B，更 C"
- "A 也好，B 也好，C 也好"
- "我们要 A，我们要 B，我们更要 C"

**为什么病**：三件套把信息密度稀释成原来的 1/3。三句里通常只有一句有用，另两句是结构填充。

**改写方法**：
1. 保留三句中最有力的一句，删其他两句
2. 三句意思真的不同 → 拆成三段，每段把这一点讲透
3. 改成非排比的不同句式表达

> **原**：写作是一种思考，是一种表达，更是一种修行。  
> **改**：写作就是思考的脚手架。

> **原**：我们要敬畏用户，要敬畏数据，更要敬畏常识。  
> **改**：用户反馈不要急着反驳，先认真看数据，再用常识检验。（三句话还在，但句式不重复）

---

## 2. 二元对立 / 否定先行

"不是 X，而是 Y" 这类结构是英文版 stop-slop 也提到的 binary contrast，在中文 AI 里同样高频。

**模板**：
- "不是 X，而是 Y"
- "X 不仅仅是 X，更是 Y"
- "与其 X，不如 Y"
- "重要的不是 X，而是 Y"
- "真正的 X 不是 A，而是 B"

偶尔用一次是修辞，连续用就是套路。一篇文章里出现两次以上就要警惕。

**改写**：直接说 Y 是什么，不用否定 X 做铺垫。

> **原**：成功不是结果，而是过程。  
> **改**：把"做完"当作终点的人，往往在第三步就放弃了。

---

## 3. 名词化动词 + 抽象搭子

把动作变成"动词 + 名词"组合。中文比英文更容易染上这毛病，因为中文动词加"进行/做出/实现"就显得正式。

**典型搭配**：

| 名词化写法 | 改写 |
|---|---|
| 进行 + 优化 / 改进 / 分析 / 沟通 | 优化 / 改 / 分析 / 聊 |
| 做出 + 选择 / 决定 / 努力 / 贡献 | 选 / 决定 / 努力 / 做了什么具体的事 |
| 实现 + 增长 / 突破 / 转型 | 涨了 / 突破了什么 / 转去做了什么 |
| 采取 + 措施 / 行动 / 策略 | 怎么做（写具体动作） |
| 加以 + 改进 / 解决 / 利用 | 改 / 解决 / 用 |
| 给予 + 支持 / 关注 / 帮助 | 支持 / 关注 / 帮忙 |
| 起到 + 作用 / 效果 | 做了什么 |
| 发挥 + 作用 / 优势 | 同上 |
| 形成 + 局面 / 态势 / 格局 | 写出实际局面 |

**整段对比**：

> **原**：我们对系统的性能进行了优化，对用户的反馈做出了响应，并对相关问题加以解决。  
> **改**：我们把首屏加载砍了 3 秒，把用户提的 17 条反馈过了一遍，关掉了 5 个老 bug。

---

## 4. 长定语 + "的"字串

中文 AI 喜欢往主语前堆一长串"的"字定语。

**模板**：
- "一个能够 XX 的 XX"
- "一种基于 XX 的、面向 XX 的、用于 XX 的 XX"
- "在 XX 背景下、由 XX 推动的、具有 XX 特征的 XX"

判断标准：一个名词前的"的"字定语超过 2 层，就要拆。

> **原**：这是一个面向中小企业、能够帮助用户在复杂业务场景下高效完成数据分析任务的工具。  
> **改**：这工具给中小企业用，主要功能是做数据分析。复杂场景跑得动，操作不需要培训。

---

## 5. 抽象主语（虚化主语）

让"时代 / 科技 / AI / 行业 / 未来 / 历史 / 现实" 做主语，听起来宏大其实是逃避——逃避交代到底是谁干的。

**典型句**：
- "时代的浪潮要求我们 XX"
- "科技正在改变我们的生活"
- "AI 正在重塑工作方式"
- "现实告诉我们 XX"
- "行业的发展呼唤着 XX"
- "未来属于 XX"
- "历史选择了 XX"
- "数据显示 / 数据告诉我们"（弱化版，但仍要警觉）

**改写原则**：把抽象主语替换成具体的人 / 团队 / 数字。

> **原**：AI 正在重塑软件开发。  
> **改**：我们组三个工程师，过去半年有两个把日常调试丢给了 Claude。

> **原**：时代要求每个人都成为终身学习者。  
> **改**：我上家公司四年招进来的同事，没自学过新东西的，到第三年基本都被边缘化了。

---

## 6. 戏剧性短句碎片

故意把一句话切成几个独立短句，制造节奏。英文叫 dramatic fragmentation。

**模板**：
- "[名词]。就这么简单。"
- "这就是答案。"
- "记住这一点。"
- "[一个词]。仅此而已。"
- "想清楚了吗。"

中文里偶尔用一次有节奏感，但 AI 喜欢段尾用、每段都用，就是套路。

**改写**：合并成一句完整的话。

> **原**：写作就是思考。仅此而已。  
> **改**：写作就是把思考写下来。

---

## 7. 反问句空转

用反问代替陈述，故作启发但其实没说出更多信息。

**模板**：
- "这难道不是 XX 吗？"
- "我们怎么能 XX 呢？"
- "谁不希望 XX？"
- "你是否也曾 XX？"
- "为什么 XX？因为 XX。"（自问自答）
- "What if XX？" 的中文翻译版：「如果我告诉你 XX 呢？」

**改写**：把反问改成陈述，直接说结论。

> **原**：这难道不是每个产品经理的梦想吗？  
> **改**：这是产品经理想要的。（或者更具体：这能省掉每周两小时的对齐会议。）

---

## 8. 立 flag 式金句收尾

中文 AI 段落收尾最稳的病。每段都试图升华，每段都来一句"准格言"。

**典型句式**：
- "这，就是 XX 的力量"
- "这，就是 XX 的意义"
- "唯有 XX，方能 XX"
- "或许，XX 才是真正的 XX"
- "XX 永远在路上"
- "致敬每一位 XX"
- "未来，属于那些 XX 的人"

**判断**：把段尾这句删了，段落还成立吗？成立就删。

人写文章不会每段都升华。让段落正常结束。

## 9. 总分总八股结构

整篇文章的骨架病。

**典型骨架**：
```
开篇：背景 + 定义 + 重要性
分点 1：首先 ……
分点 2：其次 ……
分点 3：再者 ……
（可选）分点 4：最后 ……
结尾：综上所述 / 让我们一起 / 升华金句
```

这是高考议论文 / 公文 / PPT 大纲的结构，**不是自然的文章结构**。

**可替代骨架**：

- **直接开干**：第一句就是核心结论或具体冲突，后面是论据 / 故事
- **倒三角**：最反常识 / 最重要的放最前，往后越来越细
- **故事线**：按时间 / 因果 / 冲突推进，不分点
- **单点深挖**：一篇只讲一件事，把它讲透，不凑数

如果分点结构无法避免（如说明文），至少：
- 每一点的开篇句长得不一样，不全是"首先 / 其次"
- 不要在最后一段加"综上所述"
- 不要给文章加"金句结尾"

## 10. 价值判断饱和

每个句子都在评价"这很重要 / 这很关键"，但从不说为什么、对谁。

**典型**：
- "XX 至关重要"
- "XX 是不可或缺的"
- "XX 起到了关键作用"
- "XX 具有深远意义"
- "XX 是 XX 的核心"

判断标准：评价词后面是否跟具体场景？没有就删评价、要么补具体。

> **原**：用户反馈对产品迭代至关重要。  
> **改**：上次重做支付页是因为 30 多个用户在客服那骂卡住了——反馈就是这么用的。

---

## 11. 假亲切叙述视角

强行模拟"朋友谈心"的口吻，但 AI 模拟得不准，会出现：

- 用"朋友们 / 家人们 / 宝子们 / 各位"开头
- 用"我懂你的感受 / 你不是一个人 / 你值得被看见"安慰
- "亲爱的读者，今天我想和你聊聊"
- 突然插入"对了 / 顺便说一下"假装随性

真人写谈心文章是有具体的"我"和具体的"你"——具体的经历、具体的对话、具体的场景。AI 写的"亲切"是没有锚点的空亲切。

**改写**：要么去掉假亲切称呼，纯陈述；要么补具体的人和事来支撑亲切感。

---

## 12. 远距离叙述者

英文 stop-slop 里叫 narrator-from-a-distance。中文版表现为：

- 全篇没有"我"，也没有具体的人物
- 全是"人们 / 大家 / 我们 / 用户 / 读者"这类集合名词做主语
- 描述事情用上帝视角，永远是观察者，永远不在现场

> **原**：人们在面对压力时，往往会做出不理智的选择。  
> **改**：上周我连续加班四天后，把保温杯里的水倒进了笔记本键盘——我以为那是垃圾桶。

把"人们"换成具体的"我"或"具体的某个人"，文本立刻有重量。

---

## 13. 节奏单调 / 句长趋同

AI 中文输出的隐性指纹：句子长度方差小，每句字数差不多（一般在 15–25 字区间）。

**自测方法**：把每句的字数列出来。如果连续 5 句的字数都在 ±5 字以内，节奏就是 AI 节奏。

**改写**：故意打破。把两句合并成长句，或者把一句拆成两个短句（不是戏剧性碎片，而是自然停顿）。

> **原**（每句 18 字左右）：他打开电脑准备开始工作。屏幕上显示着昨天未完成的代码。他想了想决定从头开始写。  
> **改**：他打开电脑，发现昨天的代码自己看了三分钟都没看懂——重写。

---

## 14. 三项列表偏好

只要列点，AI 倾向于列三点。两点显得不够全，四点显得太多。

如果你发现自己每次列点都列三点，停下来想：

- 真的有三点？还是凑出来的三点？
- 如果只有两点，就列两点
- 如果有四点或更多，就列四点或更多
- 如果一点能讲清楚，根本别列点

---

## 15. 每段同等长度

段落长度趋同也是 AI 指纹。AI 输出的段落普遍在 80–150 字，很少有特别短（一行）或特别长（一屏）的段落。

**真人写作的段落**会有节奏波动：
- 一句话独立成段，用来强调或转折
- 一个段落 200+ 字，深入讲一件事
- 短段 + 长段交替

**改写**：检查段落长度方差。如果全篇段落长度都差不多，主动把某些段落合并或拆开。

## 16. 标题套路（小标题指纹）

正文逐句都干净，但**一排小标题用同一个模子刻出来**，照样一眼是 AI。标题层的整齐度比正文更扎眼，因为读者扫一遍目录就感觉到了。技术文档、产品介绍、长博客最容易中招。

**三种典型标题模具：**

| 模具 | 例 | 问题 |
|---|---|---|
| `名词：一句点题` | 工作原理：全程在你本机 / 反检测：不打补丁就没痕迹 / 静默运行：绝不抢前台 | 单个没问题，**通篇一半标题都这样**就成了机器节奏 |
| 反问句标题 | 为什么传统方案这么难用 / 它凭什么不一样 / 那 XX 呢？ | 偶尔一个有劲，连着用就是套路 |
| 二元对立标题 | 为什么是 A，而不是 B | 和正文里的"不是 X 而是 Y"同源（见第 2 类），标题里更显眼 |

**判断标准**：把所有小标题抄到一起，列成一张目录。如果**超过一半是同一个句式**（都带冒号、都是反问、都是对立），就是标题指纹。

**改法**：打散模具，让标题长短和句式都不一样。

> ❌（全冒号）：工作原理：全程在本机 · 反检测：不打补丁 · 静默运行：绝不抢前台
> ✅（混着来）：全程在你本机，不走网络 · 不打补丁，也就没痕迹可查 · 它不会抢你的前台

一篇里冒号副标题留一两个无妨，反问标题最多一个，二元对立标题尽量改成陈述。关键是**别让目录读起来像一台机器列的提纲**。

---

## 使用说明

这 16 条里，**1（三件套排比）、3（名词化）、5（抽象主语）、8（金句收尾）、9（八股结构）**是中文 AI 文本最强的五个指纹。改写时优先处理这五项，命中率最高。**16（标题套路）**是长文 / 技术文档专属的高频指纹，正文干净也要单独查一遍标题。

其余十项是补充指纹，遇到就处理；找不到不必强求。
