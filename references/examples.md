# 改写示例：Before / After

四个典型场景。每例都标注改了什么、为什么。原文是真实 AI 生成（或贴近 AI 输出的稿）的样子，改写示范"去 AI 味"的具体动作。

---

## 场景一：公众号 / 知乎长文

### 例 1.1 · 科技评论

**Before（AI 味浓）：**

> 在当今人工智能蓬勃发展的时代，AI 正在以前所未有的速度重塑我们的工作方式。从内容创作到代码编写，从数据分析到决策辅助，AI 的触角已经延伸到我们生活的方方面面。
>
> 不可否认，这是一场深刻的变革。它不仅仅是工具的升级，更是思维方式的革命，是人类文明发展的又一个里程碑。
>
> 然而，值得注意的是，技术的进步从来都不是一帆风顺的。我们必须认识到，机遇与挑战并存，蓝海与暗礁同在。唯有保持敬畏之心，方能在 AI 浪潮中行稳致远。

**问题诊断：**
- 第一段：抽象主语「AI」做事，三件套排比「从…到…，从…到…」
- 第二段：「不仅仅是…更是…是…」三件套 + 抽象升华
- 第三段：「不可否认 / 我们必须认识到 / 唯有…方能…」全是空壳，金句结尾
- 全篇没有一个具体的人、一个具体的数字、一个具体的场景

**After：**

> 我们组三个工程师，过去半年有两个把日常调试丢给了 Claude。剩下那个还在用断点法，每次 review 都比另外两个慢半天。
>
> 去年说"AI 写代码"听起来还像段子，现在再说就是一句陈述句。变得太快——三月份我刚学的 prompt 技巧，到十月份模型自己就懂了，技巧白学。
>
> 但写代码不是全部。我家楼下打印店老板用 AI 写公众号，三个月涨了 2000 粉，比他自己写一年都快。他不懂"AI 时代"是什么，他只看后台数据。

**改了什么：**
- 删了所有「在 XX 时代 / 浪潮 / 里程碑 / 行稳致远」类抒情
- 抽象主语「AI」换成具体人物（同事、打印店老板）
- 拆掉所有三件套
- 给了三组具体数字（3 个 / 2 个 / 半年 / 2000 粉 / 3 个月）
- 段落长度刻意不同（短-中-中）
- 没有金句收尾

### 例 1.2 · 职场鸡汤类

**Before：**

> 在这个快节奏的时代，焦虑似乎成了每个人的标配。我们焦虑工作，焦虑收入，焦虑未来。其实，焦虑的本质，是对失控的恐惧。
>
> 那么，我们该如何应对焦虑呢？首先，要学会与不确定共处；其次，要建立自己的核心能力；最后，要保持对生活的热爱。
>
> 愿你在喧嚣的世界里，找到属于自己的节奏。愿你不被焦虑裹挟，做时间的朋友，做自己的主人。

**问题诊断：**
- 「在 XX 时代」开场
- 「焦虑 A，焦虑 B，焦虑 C」三件套
- 「焦虑的本质，是 XX」AI 最爱的「XX 的本质是 XX」句式
- 反问句「我们该如何 XX 呢？」自问自答
- 「首先 / 其次 / 最后」八股
- 「愿你 / 愿你 / 做 XX 的朋友 / 做 XX 的主人」金句收尾 + 三件套

**After：**

> 我上次焦虑发作是上个月，凌晨三点，原因是想到房贷还有 18 年。
>
> 翻身起来做了三件事：把工资条找出来核了一遍存款；查了如果裸辞够撑几个月（答案：7 个月）；列了一份"明天要做的事"，三条。做完这三件事，我睡着了。
>
> 这套不能治本，但能让我从"完蛋了"变成"那现在干什么"。够用。

**改了什么：**
- 删开场背景，直接进具体场景（凌晨三点 / 18 年）
- 拆「焦虑 A B C」排比
- 「我们该如何 XX」反问 → 直接说"做了三件事"
- 删「首先 / 其次 / 最后」八股，用动词序列代替
- 删「愿你 / 愿你」金句结尾
- 末段是结论，不是升华

## 场景二：小红书 / 短文案

### 例 2.1 · 产品推荐

**Before：**

> 姐妹们！今天必须给大家安利一款宝藏护肤品！🌟
>
> 这款精华真的太太太爱了！它不仅能深层补水，还能淡化细纹，更能提亮肤色，是一款集多重功效于一体的全能型选手！
>
> 用了一个月，皮肤状态明显改善，整个人都焕然一新！这就是好产品的力量！宝子们冲就完事了！

**问题诊断：**
- 「姐妹们 / 安利 / 宝藏 / 宝子们」高度套路化（这就是小红书 AI 文案的指纹）
- 「不仅 A，还 B，更 C」三件套
- 「全能型选手」AI 套词
- 「明显改善 / 焕然一新」空泛形容
- 「这就是 XX 的力量」金句收尾
- 「冲就完事了」AI 模仿网感词的常见误用

**After**（保留小红书风但去掉 AI 指纹）：

> 用了 28 天，记录在手机备忘录里。
>
> 用之前：T 区出油到中午要补妆，左脸颊有一片粗糙摸起来像砂纸。
> 用之后：T 区还是出油但补妆从中午延到下午 4 点，砂纸感的位置肉眼看不出来了，摸还有一点。
>
> 一瓶 380，30ml，省着用够两个月。不踩雷。

**改了什么：**
- 删「姐妹们 / 宝子们 / 安利 / 宝藏」全部套路称呼和热词
- 删 emoji（AI 加 emoji 的位置很规律）
- 拆「不仅 A 还 B 更 C」
- 把「明显改善」换成具体观察（T 区 / 中午 / 4 点 / 砂纸感）
- 给具体数字（28 天 / 380 / 30ml）
- 删金句结尾

**注意**：小红书本身是高度套路化的平台，完全去套路就不像小红书了。这里的目标是"保留小红书的具体感和实测感，去掉 AI 模仿小红书的尴尬感"。

### 例 2.2 · 探店

**Before：**

> 探店日记 | 一家藏在巷子里的宝藏咖啡馆 ☕
>
> 推开门的瞬间，仿佛进入了另一个世界。原木色调的装修温馨而治愈，氤氲的咖啡香扑面而来，让人瞬间忘却城市的喧嚣。
>
> 这里的每一杯咖啡都是匠心之作，是咖啡师对咖啡的热爱，是一种生活态度的表达，更是一份与城市对话的温柔。
>
> 在这个浮躁的时代，能有这样一个角落，真的太治愈了。强烈推荐！

**问题诊断：**
- 「藏在巷子里的宝藏」AI 探店专用套句
- 「仿佛进入另一个世界 / 忘却喧嚣」陈词比喻
- 「氤氲」AI 偏爱的"文艺"词
- 「是 A，是 B，更是 C」三件套
- 「在这个 XX 的时代」+ 「真的太 XX 了」+ 「强烈推荐」三连套话

**After：**

> 探店 | 长寿路 487 弄拐进去第二家
>
> 老板自己烘豆，店里只有六个位子。我去的下午两点没人，三点开始陆续来人，到四点要拼桌。
>
> 点了哥伦比亚水洗手冲，48 块。豆子刚开封三天，喝得出花香没有酸。配了一块巴斯克，蛋香盖过焦味，不甜。
>
> 缺点：洗手间要去隔壁面馆借。

**改了什么：**
- 删「藏在巷子里」「宝藏」开篇套话，换成具体地址
- 删比喻「另一个世界 / 忘却喧嚣」，换成观察（六个位子 / 两点没人 / 三点来人）
- 拆「是 A，是 B，更是 C」
- 删「氤氲 / 治愈 / 浮躁的时代」抒情
- 给具体的咖啡名、价格、品鉴细节
- 加一条缺点——AI 几乎从不主动写缺点，写了缺点立刻像真人

## 场景三：公文 / 工作汇报（边界场景）

公文有结构要求，"总分总 / 首先其次"不全是 AI 味，是格式要求。这里示范的是**在保留公文规范的前提下**去掉 AI 痕迹（多余的抒情和价值判断饱和）。

### 例 3.1 · 项目周报

**Before：**

> 一、本周工作概述
>
> 本周，在领导的关心和支持下，在各部门的密切配合下，项目组全体成员凝心聚力、攻坚克难，圆满完成了既定目标。我们以高度的责任感和使命感，扎实推进各项工作，取得了显著成效。
>
> 二、主要进展
>
> 首先，我们对系统架构进行了优化；其次，我们对用户反馈做出了响应；再者，我们对相关 bug 加以解决。
>
> 三、下周计划
>
> 下周，我们将再接再厉，砥砺前行，以更加饱满的热情投入工作，为项目的顺利推进贡献力量。

**问题诊断：**
- "在领导的关心和支持下" 这类政治套话——如果是党政公文这是规范，如果是企业内部周报就是 AI 自动加的
- 「凝心聚力 / 攻坚克难 / 责任感和使命感」成语堆砌
- 「显著成效」无具体
- 「进行了优化 / 做出了响应 / 加以解决」名词化三连
- 「再接再厉 / 砥砺前行 / 饱满的热情 / 贡献力量」金句结尾全套

**After**（企业内部周报版本）：

> **本周进展**
>
> 1. 把订单查询接口的响应时间从 1.2s 降到 280ms。瓶颈在 N+1 查询，加了批量预加载。
> 2. 处理了上周用户反馈的 12 条问题，其中 9 条已修复，2 条转产品评估，1 条无法复现。
> 3. 修了三个线上 bug：登录态丢失、列表分页错位、价格四舍五入偏差。
>
> **下周**
>
> 重写支付回调的幂等逻辑。预计三天。如果遇到第三方文档不清楚的地方可能延到周五。
>
> **风险**
>
> 测试环境数据库下周二要迁移，可能影响联调。已和测试组对齐。

**改了什么：**
- 删开场客套
- 把「进行优化 / 做出响应 / 加以解决」全部去名词化，换成具体动作 + 数字
- 删「再接再厉 / 砥砺前行」收尾，换成具体下周计划
- 主动加「风险」段——真实的工作汇报会有风险/阻塞，AI 生成的只会画饼

## 场景四：学术 / 研究类（边界场景）

学术写作允许被动语态、抽象主语、长句——但 AI 写的学术文本仍有可识别的"伪学术"指纹：过度概念化、空泛价值判断、引用堆砌但不深入。

### 例 4.1 · 论文摘要

**Before：**

> 随着人工智能技术的飞速发展，大语言模型在自然语言处理领域展现出了巨大的潜力。本文旨在探讨大语言模型在文本生成任务中的应用现状与未来发展趋势。
>
> 通过对相关文献的梳理与分析，本文发现，当前大语言模型在文本生成方面取得了显著进展，但仍面临诸多挑战。这些挑战既有技术层面的，也有伦理层面的，更有社会层面的。
>
> 本文认为，未来的研究应当注重多维度的协同发展，以期推动该领域的持续繁荣。

**问题诊断：**
- 「随着 XX 的飞速发展 / 巨大的潜力」开场套话
- 「旨在探讨 XX 的现状与未来发展趋势」AI 摘要标准句
- 「显著进展 / 诸多挑战」无具体
- 「既有 A，也有 B，更有 C」三件套
- 「应当注重多维度的协同发展，以期推动该领域的持续繁荣」纯空话收尾

**After：**

> 本文对 2023–2025 年间 47 篇 LLM 文本生成相关论文做了系统综述，重点检查三个指标：评估方法、人类偏好对齐、长文本一致性。
>
> 主要发现：(1) 80% 的论文用 BLEU/ROUGE 作为主要指标，但这两个指标与人类评分相关系数仅为 0.31。(2) RLHF 类方法在短文本上效果稳定，但在超过 2000 token 的生成任务中崩塌率高于纯监督微调。(3) 评估数据集严重偏向英文，中文长文本评估缺乏统一基准。
>
> 据此提出三点建议：放弃 BLEU 作为主指标；构建中文 4k+ token 评估集；分离短文本与长文本基准。

**改了什么：**
- 删「随着 XX 的发展」开场
- 「本文旨在」→ 直接说做了什么（综述 47 篇 / 2023–2025 年）
- 「显著进展 / 诸多挑战」→ 三条具体发现，带数字
- 删「既有 A 也有 B 更有 C」
- 「应当注重多维度的协同发展」→ 三条具体建议
- 摘要长度差不多，但信息密度从 0 变成可引用

## 通用改写流程（看完示例后的总结）

每次改写按这个顺序过，命中率最高：

1. **删开场背景**（"在 XX 时代 / 随着 XX 发展"）
2. **找抽象主语**，换成具体的人 / 团队 / 数字
3. **拆三件套排比**（搜「，是…，是…，更是…」/「既要…也要…还要…」）
4. **去名词化**（搜「进行 / 实现 / 做出 / 采取 / 加以 / 给予」）
5. **删价值判断饱和词**，每一个都问"能换成具体吗"
6. **删段尾金句**（"这就是 XX 的力量"等）
7. **删元评论**（"接下来 / 让我们 / 在本文中"）
8. **检查段落长度方差**，过于均匀就主动打乱

按这个流程过一遍，普通 AI 输出能从 25/50 提到 38/50 左右。剩下的提升靠加具体内容——AI 不知道的、只有写作者本人才知道的细节。

---

## 最重要的一句

**消除 AI 味的本质不是"换词"，是"给具体"。**

任何抽象、任何套话、任何升华——背后都是 AI 因为不知道细节而不得不抽象。你知道细节，所以你能写得不像 AI。把你知道的细节写进去，AI 味自然就没了。
